# Сервис управления рассылками

Веб-приложение для создания и управления email-рассылками с поддержкой ролей, статистикой и кешированием.

## Функциональность

- **Управление клиентами** (получателями): добавление, редактирование, удаление.
- **Управление сообщениями**: тема и тело письма.
- **Управление рассылками**: дата начала/окончания, статус (Создана/Запущена/Завершена), выбор сообщения и получателей.
- **Отправка рассылок** «по требованию» через `send_mail()` с сохранением попыток (пакетная запись).
- **Регистрация и аутентификация** пользователей с подтверждением email, восстановлением пароля.
- **Роли и права доступа**:
    - Пользователь → управляет только своими объектами.
    - Менеджер → просмотр всех рассылок/клиентов, список пользователей, блокировка, отключение рассылок.
    - Администратор → полный доступ ко всем действиям.
- **Статистика** по успешным/неуспешным попыткам, количеству отправленных сообщений.
- **Кеширование** главной страницы и статистики через Redis.

## Технологии

- Python 3.13, Django 6.0
- PostgreSQL (psycopg2)
- django-allauth (аутентификация)
- Redis + django-redis (кеширование)
- Bootstrap 5 (фронтенд)
- dotenv для переменных окружения

## Требования

## Установка и настройка

1. **Клонировать репозиторий**
   ```
   git clone <repo-url>
   cd SkyPro_course_project_4
   ```
Создать и активировать виртуальное окружение

```
python -m venv .venv
.venv\Scripts\activate      # Windows
source .venv/bin/activate   # Linux/Mac
```
Установить зависимости
```
pip install -r requirements.txt   # или через poetry
poetry install
```
Настроить переменные окружения

Создать файл .env в корне проекта по примеру .env.example:

```
SECRET_KEY=ваш-секретный-ключ
DEBUG=True
NAME=имя_базы_данных
USER=пользователь_БД
PASSWORD=пароль_БД
HOST=localhost
PORT=5432
```
Применить миграции

```
python manage.py migrate
```
Запустить Redis (локально на 127.0.0.1:6379)

Запустить сервер

```
python manage.py runserver
```
## Использование

1. Зарегистрируйтесь на сайте (ссылка подтверждения отправляется на email, письма отображаются в консоли в формате base64).
2. Войдите в систему.
3. Создайте получателей, сообщения, затем рассылку (укажите дату старта, финиша, выберите сообщение и получателей).
4. Когда рассылка перейдёт в статус «Запущена», появится кнопка «Отправить сейчас».
5. Статистика по попыткам доступна в разделе «Статистика».

Администратор (суперпользователь) и менеджеры могут управлять пользователями.

## Роли
* Пользователь – работа только со своими клиентами, сообщениями и рассылками.
* Менеджер – просмотр всех данных, блокировка пользователей, принудительное завершение рассылок.
* Администратор – полные права на все действия.

Назначать менеджеров может администратор на странице «Пользователи».
